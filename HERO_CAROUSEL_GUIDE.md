## How It Works

### HTML Structure

```
.hero
  └── .hero-bg #heroCarousel
        ├── .hero-slides
        │     ├── .hero-slide .hero-slide--active  (slide 1, shown)
        │     ├── .hero-slide                       (slide 2, hidden)
        │     └── ... × 20
        ├── .hero-overlay   (gradient, unchanged)
        └── .hero-grain     (grid texture, unchanged)
  └── .hero-dots            (dot nav, appended below hero-bg)
  └── .hero-progress        (progress bar, injected by JS)
  └── .hero-content         (text + CTAs, unchanged)
  └── .hero-torn            (fade to black, unchanged)
```

### CSS System

- `.hero-slide` — `position: absolute; inset: 0; opacity: 0; transition: opacity 1.4s`
- `.hero-slide--active` — `opacity: 1`
- Ken Burns: `@keyframes heroKenBurns` — subtle 1.05→1.0 scale on the active image, re-triggered on each slide change
- Progress bar: `.hero-progress--running` drives a 6s linear width transition (0%→100%)
- Dots: pill-shaped active dot (24px wide), circle inactive (8px)
- `@media (prefers-reduced-motion: reduce)` — disables all transitions and animations

### JS Engine (`CAROUSEL` object)

| Method | Purpose |
|--------|---------|
| `CAROUSEL.init()` | Reads slides, builds dots + progress bar, attaches events |
| `CAROUSEL.goTo(index)` | Cross-fades to slide N, updates dots, re-triggers Ken Burns |
| `CAROUSEL.start()` | Starts 6-second `setInterval`, kicks off progress bar |
| `CAROUSEL.pause()` | Clears timer, freezes progress bar at current width |
| `CAROUSEL.resume()` | Restarts from current position |
| `CAROUSEL.restart()` | pause + start with 100ms buffer |
| `CAROUSEL.next()` | Advance one forward |
| `CAROUSEL.prev()` | Step one back |
| `CAROUSEL.initSwipe()` | Touch start/end delta, 50px threshold, directional guard |

## Behavior Matrix

| Trigger | Action |
|---------|--------|
| Auto (6s) | Advance to next slide |
| Mouse hover on hero | Pause carousel + freeze progress |
| Mouse leave hero | Resume from current slide |
| Dot click | Jump to slide + restart timer |
| Swipe left | Next slide + restart timer |
| Swipe right | Previous slide + restart timer |
| Arrow key (focused) | Left/Right advance |
| Tab hidden | Pause (Page Visibility API) |
| Tab visible | Resume |
| `prefers-reduced-motion: reduce` | No transitions, no timer, no Ken Burns |

## Performance Notes

- Images 1–3 load eagerly (`loading=""`) — above fold, critical
- Images 4–20 load lazily (`loading="lazy"`) — defer until needed
- `will-change: opacity` on `.hero-slide` — hints GPU compositing
- Only `transform` and `opacity` are animated — no layout thrash
- Progress bar uses CSS `transition: width` — single composited property
- `{ passive: true }` on all scroll/touch listeners

## Adding / Removing Images

**To add a new slide:**
1. Place the image in `assets/hero/food-carousel/`
2. Add a new `.hero-slide` div in `index.html` inside `.hero-slides`
3. Add a dot will be auto-generated by JS — no changes needed

**To remove a slide:**
1. Delete the `.hero-slide` div from `index.html`
2. JS rebuilds dots on init — no other changes needed

**To change the interval:**
In `script.js`, change: `interval: 6000` (ms)
